# User Research and Validation

## Overview
Perform customer interviews and validation sprints to verify feature assumptions before writing code.

## When to Use
- Validating a new startup idea.
- Planning major product modifications or updates.
- Investigating customer churn.

## Instructions
1. Select 5 active or churned users for interviews.
2. Ask open-ended questions about user workflows rather than pitching solutions.
3. Track patterns in user complaints or request trends.
4. Run validation tests (such as a fake door button) to measure concrete demand.
