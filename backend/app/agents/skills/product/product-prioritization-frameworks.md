# Product Prioritization Frameworks

## Overview
Deploys frameworks to evaluate and rank feature requests, bugs, and improvements based on impact, resources, and evidence.

## When to Use
- Determining which feature to build next.
- Resolving conflicting inputs from team members or customers.
- Managing product backlogs.

## Instructions
1. Apply **RICE** score to proposed features:
   - `Reach` (Number of users affected)
   - `Impact` (Individual value addition)
   - `Confidence` (Available validation data)
   - `Effort` (Development time)
   - `RICE = (Reach * Impact * Confidence) / Effort`
2. Sort features by RICE score to focus on high-leverage deliverables.
