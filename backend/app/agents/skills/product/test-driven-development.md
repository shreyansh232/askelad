# Test-Driven Development (TDD)

## Overview
Develop features systematically by writing failing unit or integration tests before implementing source code.

## When to Use
- Writing critical backend routing or service logic.
- Refining complex business logic operations.
- Building reliable refactoring pipelines.

## Instructions
1. Write a simple, failing test defining the expected behavior.
2. Implement the minimal codebase changes necessary to make the test pass.
3. Refactor the code for clean styling and performance while ensuring tests remain green.
